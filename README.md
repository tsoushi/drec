# drec

飲んだ薬をその場でサッと記録する、個人用の服薬記録 Web アプリ。

## スタック

- React Router v7 (framework mode) + React 19 + TypeScript
- Vite / Tailwind CSS v4
- SQLite（better-sqlite3、素の SQL）

## セットアップ

```sh
npm install
```

## 開発

```sh
npm run dev
```

→ http://localhost:5173 （ポートを変える場合は `npm run dev -- --port 3001`）

## 本番

```sh
npm run build
npm run start
```

→ http://localhost:3000

## 型チェック

```sh
npm run typecheck
```

## データ

- DB ファイル: `data/drec.db`（WAL モード。`data/` は起動時に自動生成）。
- 保存先を変えたい場合は環境変数 `DREC_DB` を指定（例: `DREC_DB=C:\path\to\drec.db`）。

### 記録項目

薬剤名（必須） / 製品名 / 量（数値＋単位） / 服用時刻 / 時刻の誤差（±分・任意） / ピーク(分)（任意・グラフ用） / 備考 / 作成時刻（変更不可） / 更新時刻。

### コメント

服薬記録とは別に、コメントを同じ時系列に残せます。コメントは0件以上の対象を「メンション」（参照）でき、対象は**記録・コメント・メンタル**の3種です。記録/メンタルカードの「コメント」ボタン、コメントカードの「参照」ボタンから、対象を参照した状態で書けます（複数参照も可）。コメントのメンションチップをタップすると対象までスクロール＆ハイライト。コメントも時刻（手入力切替可）・時刻の誤差（±分）・編集・論理削除に対応。テーブルは `comments` と多対多の `comment_mentions`（→記録）/ `comment_comment_mentions`（→コメント）/ `comment_mental_mentions`（→メンタル）。

### メンタル記録

いまのメンタルを **-10〜10** の値で記録できます（フォーム上部の「メンタル」タブ、スライダーで入力）。記録・コメントと同じく **記録時刻**（手入力切替可）・**時刻の誤差（±分）**・編集・論理削除に対応し、同じ時系列（タイムライン）に並びます。コメントから参照（メンション）できます。テーブルは `mentals`。血中濃度グラフに重ねて表示できます（下記）。

### 経過時間の表示

記録・コメント・メンタルは、日時の右に現在時刻との差（過去はマイナス、例: `-3d2h30m`）を表示します（1秒ごとにリアルタイム更新）。コメントのメンションには、対象との時間差（例: `+2h30m`）を常に表示します。

### 削除について

削除は**論理削除**です。アプリの一覧からは消えますが、行は DB に残り `deleted_at` に削除時刻が
入ります。削除済みの確認・復元は DB を直接操作してください。

```sql
-- 削除済みを見る
SELECT * FROM records WHERE deleted_at IS NOT NULL;

-- 復元する
UPDATE records SET deleted_at = NULL WHERE id = ?;
```

### 血中濃度グラフ

画面右上の「グラフ」リンク（`/graph`）。登録済みの記録（量入力あり）から、ごく簡易なモデルで血中濃度を推定してグラフ表示します（読み取り専用）。

- モデル: 単位服用量で血中濃度 +1。服用からピークまで直線上昇、以降は半減期で指数減衰。複数服用は足し合わせ
- ピークまでの時間は、レコードに「ピーク(分)」が設定されていればそれを優先し、未設定のレコードは画面の既定値（薬剤ごとに保存）を使用
- 既定値: 単位服用量 10 / ピーク 25分 / 半減期 60分 / 表示範囲 ±72時間（すべて画面上で変更可）
- **複数の薬剤を同時表示**: トグルチップで薬剤を複数選択でき、それぞれ固有色の線として重ねて比較表示（各薬剤は自身の保存済み設定で計算・合算はしない）。凡例に薬剤名と基準時刻の推定濃度を併記。パラメータは「編集対象」で選んだ 1 薬剤について編集
- 基準時刻（グラフ中央）はドラッグ / スワイプで移動可。現在の基準時刻をテキスト表示し、「今に戻す」で現在時刻に復帰（追従再開）。実際の現在時刻はオレンジの「今」線で表示
- 横軸の目盛りは実時刻（時刻 / 日付ラベル）に固定されており、ドラッグすると曲線と一緒にスライドする
- 設定（単位服用量・ピーク・半減期・期間）は薬剤ごとに DB（`graph_settings` テーブル）に保存され、切り替えやリロードでも復元される
- 選択中の薬剤は URL のクエリパラメータ（`?drug=薬剤名`、複数可）に保持され、リロードやブックマークで復元される
- 曲線上に点を表示: **服用**（薬剤色・服用時刻）と**コメント**（青・コメント時刻）。タップ / クリック / フォーカスで詳細ポップオーバー（服用: 薬剤名・製品名・時刻・±誤差・量・ピーク・備考 / コメント: 本文・時刻・±誤差・選択中各薬剤のその時点の推定濃度）。コメントを効果予測グラフに照らして「この時点で吐き気」等を分析できる
- **メンタルの重ね表示**: 「その他」の「🧠 メンタル」トグルで、メンタル記録（-10〜10）を薬剤とは独立した**絶対目盛り**で重ねて表示できる（0=中央・+10=上・-10=下。薬剤の濃度スケールとは無関係の固定位置）。紫の線＋点で描画し、右端に +10 / 0 / -10 の目安線を表示。点をタップでメンタル値・時刻・±誤差を表示。表示状態は URL（`?mental=1`）に保持される

### 月別レポート

画面右上の「レポート」リンク（`/report`）。カレンダー月ごと（今月が先頭、新しい順）に、
飲んだ薬剤の一覧・回数・合計量（単位別）を表示します（読み取り専用）。

### 変更ログ

すべての DB 変更（作成・更新・削除）は、サーバのコンソールと `data/changes.log`（JSON Lines）に追記されます。出力先は環境変数 `DREC_LOG` で変更できます。画面右上の「ログ」リンク（`/logs`）から閲覧できます。
